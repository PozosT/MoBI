El problema del warm-up¶

Toda simulación arranca desde un estado inicial X(0) que no es representativo del estado estacionario. Las observaciones del período transitorio sesgan la estimación.

$$E\left[\bar{Y}_m\right] = \theta + \underbrace{\frac{1}{m}\sum_{i=1}^{m}(E[Y_i] - \theta)}_{\text{sesgo de inicialización}}$$

El sesgo no desaparece al aumentar réplicas. Solo se elimina descartando el transitorio.

Método de Welch: promediar entre réplicas → suavizar con media móvil → identificar estabilización visual.

Batch Means¶

Una corrida larga (post warm-up) se divide en m lotes de b observaciones. Las medias de lote Ȳ₁,...,Ȳₘ se tratan como i.i.d. si b es suficientemente grande.

$$IC = \bar{Y}_m \pm t_{\alpha/2,\, m-1} \cdot \frac{S_B}{\sqrt{m}}$$

Verificación de independencia (Von Neumann): $$VN = \frac{\sum(\bar{Y}_{j+1} - \bar{Y}_j)^2}{(m-1) \cdot S_B^2} \approx 2 \text{ bajo independencia}$$

Si VN << 2, los lotes están correlacionados → aumentar b.

import simpy
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats as st

def mm1_largo(lam, mu, n_clientes, seed):
    """Simula M/M/1 hasta atender n_clientes y retorna el vector de esperas."""
    np.random.seed(seed)
    esperas = []

    def cliente(env, srv):
        t_arr = env.now
        with srv.request() as req:
            yield req
            esperas.append(env.now - t_arr)
            yield env.timeout(np.random.exponential(1/mu))

    def llegadas(env, srv):
        i = 0
        while i < n_clientes:
            yield env.timeout(np.random.exponential(1/lam))
            env.process(cliente(env, srv))
            i += 1

    env = simpy.Environment()
    srv = simpy.Resource(env, capacity=1)
    env.process(llegadas(env, srv))
    env.run()
    return np.array(esperas)

Common Random Numbers (CRN)¶

Para comparar dos sistemas, usar la misma secuencia aleatoria en ambos. Esto introduce correlación positiva y reduce Var[Δ].

$$\text{Var}[\Delta_j] = \text{Var}[Y_j^{(1)}] + \text{Var}[Y_j^{(2)}] - 2\,\text{Cov}(Y_j^{(1)}, Y_j^{(2)})$$

Con CRN: Cov > 0 → Var[Δ] se reduce → IC más estrecho → menos réplicas necesarias.

⚠️ CRN requiere sincronización correcta: el mismo número aleatorio debe usarse para el mismo tipo de evento en ambos sistemas.

Ejemplo 2 — M/M/1 vs M/D/1 con CRN¶

Misma carga (λ=4, μ=5, ρ=0.8), diferente variabilidad de servicio:
  • Sistema A: servicio exponencial (CV=1) → Wq = 0.80 min
  • Sistema B: servicio determinista (CV=0) → Wq = 0.40 min

El servicio determinista reduce Wq exactamente a la mitad.

def mm1_vs_md1_crn(lam, mu, n_clientes, seed, warmup=1000):
    """M/M/1 y M/D/1 con CRN: misma secuencia de llegadas y mismos U_i."""
    np.random.seed(seed)
    inter_arrivals = np.random.exponential(1/lam, n_clientes)
    services_exp = np.random.exponential(1/mu, n_clientes)
    service_det = 1/mu

    def sim_queue(services, det=False):
        esperas = np.zeros(n_clientes)
        t_arr, t_sal = 0.0, 0.0
        for i in range(n_clientes):
            t_arr += inter_arrivals[i]
            w = max(0.0, t_sal - t_arr)
            esperas[i] = w
            s = service_det if det else services[i]
            t_sal = t_arr + w + s
        return esperas[warmup:].mean()

    return sim_queue(services_exp), sim_queue(None, det=True)


def mm1_vs_md1_indep(lam, mu, n_clientes, seed_A, seed_B, warmup=1000):
    """M/M/1 y M/D/1 con semillas independientes (sin CRN)."""
    def run(ia, sv, det=False):
        esperas = np.zeros(n_clientes)
        t_arr, t_sal = 0.0, 0.0
        for i in range(n_clientes):
            t_arr += ia[i]
            w = max(0.0, t_sal - t_arr)
            esperas[i] = w
            s = (1/mu) if det else sv[i]
            t_sal = t_arr + w + s
        return esperas[warmup:].mean()

    np.random.seed(seed_A)
    ia_A = np.random.exponential(1/lam, n_clientes)
    sv_A = np.random.exponential(1/mu, n_clientes)
    np.random.seed(seed_B)
    ia_B = np.random.exponential(1/lam, n_clientes)
    return run(ia_A, sv_A), run(ia_B, None, det=True)

def call_center(lam_h, mu_L, sigma_L, c, T_h, seed, warmup_h=200):
    """Simula call center M/G/c con servicio Log-normal. Tiempos en min."""
    np.random.seed(seed)
    esperas = []
    T_min = T_h * 60
    warmup_min = warmup_h * 60
    lam_min = lam_h / 60

    def cliente(env, srv):
        t_arr = env.now
        with srv.request() as req:
            yield req
            w = env.now - t_arr
            if env.now > warmup_min:
                esperas.append(w)
            yield env.timeout(np.random.lognormal(mu_L, sigma_L))

    def llegadas(env, srv):
        while True:
            yield env.timeout(np.random.exponential(1/lam_min))
            env.process(cliente(env, srv))

    env = simpy.Environment()
    srv = simpy.Resource(env, capacity=c)
    env.process(llegadas(env, srv))
    env.run(until=T_min)

    if not esperas:
        return 0.0, 0.0
    return float(np.mean(esperas)), float(np.mean([w > 2 for w in esperas]))

Conclusiones — Módulo 3¶

  • El warm-up debe descartarse explícitamente; el método de Welch lo identifica gráficamente.
  • Batch Means estima IC para sistemas estacionarios; la razón de Von Neumann valida la independencia entre lotes.
  • Common Random Numbers reduce drásticamente la varianza de Δ y permite decidir entre alternativas con menos réplicas.
  • La prueba t pareada acompaña al IC: cuantifica la evidencia con un p-valor y un tamaño de efecto.
  • Para múltiples métricas, aplicar Bonferroni o un equivalente para no inflar el error tipo I.
  • En ausencia de fórmulas (M/G/c con LogNormal, prioridades, redes) la simulación es la única vía.
